Version 1.1 | 2025.01.23

## 1. Overview

### 1.1 Product Summary
PLAM은 사용자가 TODO를 입력하면 AI가 서브태스크로 분해하고, 예상 기상 시간 ~ 취침 시간 사이의 실제 가용 시간을 기준으로 기존 일정과 이미 배치된 TODO 소요시간을 모두 고려하여 최적의 날짜에 자동 배치해주는 스마트 일정 관리 앱입니다.

### 1.2 한 줄 컨셉
“TODO 던지면, AI가 내 하루 체력 안에서 쪼개서 배치해주는 앱”

### 1.3 Core Value Proposition
- 현실적인 가용 시간 계산: 기상/취침 시간 + 고정 일정 + 기존 TODO 소요시간을 모두 반영
- AI 기반 TODO 자동 분해: 복잡한 작업을 실행 가능한 서브태스크로 분리
- 스마트 날짜 배치: 하루에 과부하되지 않도록 날짜 단위로 분산
- 통합 뷰: 고정 일정과 TODO를 하나의 캘린더에서 관리
- 동기부여 시스템: 스트릭과 잔디로 꾸준한 실행 유도

### 1.4 Target User
일정을 계획은 하지만 항상 과부하되거나, 현실적인 시간 감각 없이 TODO를 쌓아두는 대학생 및 직장인.

## 2. Problem Statement

### 2.1 기존 앱의 문제점
문제 | 설명
--- | ---
가용 시간 무시 | 하루에 실제로 쓸 수 있는 시간이 얼마인지 고려하지 않음
TODO 과적재 | 이미 많은 TODO가 있음에도 계속 추가
반복 일정의 비현실성 | 반복 TODO가 쌓이며 피로도 증가

### 2.2 PLAM의 솔루션
문제 | PLAM 솔루션
--- | ---
가용 시간 무시 | 기상~취침 사이 시간에서 모든 일정/작업을 차감
TODO 과적재 | 이미 배치된 TODO 소요시간을 포함해 재계산
반복 일정의 비현실성 | 반복 TODO를 독립적인 개별 일정으로 관리

## 3. Time Model (핵심 로직)

### 3.1 하루 가용 시간 계산
하루 총 가용 시간 = (취침 시간 - 기상 시간)
                  - 고정 일정 시간 합
                  - 이미 배치된 TODO 소요시간 합

- 가용 시간이 0 이하인 날에는 추가 TODO 배치 불가
- 이 값이 AI 분배의 최우선 기준

### 3.2 새벽 시간 처리 규칙
- 하루의 마감 기준은 다음날 새벽 03:00
- 00:00 ~ 03:00은 전날의 연장 시간으로 간주
- 해당 구간에 배치 시 반드시 경고 노출

⚠️ "해당 TODO는 새벽 시간에 배치됐어요. 괜찮으신가요?"

---

## 5. 반복 TODO 정책 (명확화)

### 5.1 기본 원칙
- 반복 TODO는 각 날짜마다 독립적인 TODO 인스턴스로 생성
- 일반 TODO와 동일하게 소요시간, 완료 여부, 스트릭에 영향

### 5.2 미완료 처리
- 미완료 시 다음 날로 미루기 제안
- 다음 반복 주기와는 완전히 분리

---

## 6. AI 분배 로직 (수정)

### 6.1 Input 확장
```json
{
  "dayContext": {
    "wakeTime": "08:00",
    "sleepTime": "23:00",
    "fixedSchedulesMinutes": 180,
    "existingTodoMinutes": 240
  }
}
```

### 6.2 분배 원칙
1. 하루 가용 시간 초과 금지
2. 마감일 우선
3. 이미 과부하된 날짜는 자동 회피
4. 불가피할 경우 새벽 배치 + 경고
